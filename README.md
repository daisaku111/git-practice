# 永久保存版：Git & GitHub 実務ワークフロー完全マニュアル

## 1. 必須用語チートシート

| 単語 | 現場での意味・使われ方 |
| :--- | :--- |
| **リポジトリ (Repository)** | プログラムと「変更履歴」を保存する箱。PC内を「ローカル」、GitHub上を「リモート」と呼ぶ。 |
| **クローン (Clone)** | 既存のプロジェクト（リモートリポジトリ）を自分のPCに丸ごとダウンロードすること。 |
| **ブランチ (Branch)** | メインのデータを壊さないように作る「作業用の分身」。メイン軸は `main` と呼ばれる。 |
| **コミット (Commit)** | PC内での「セーブ」。必ず「何の変更をしたか」がわかるメッセージを添える。 |
| **プッシュ (Push)** | PC内のセーブデータを、GitHub（インターネット上）へアップロードすること。 |
| **プル (Pull)** | GitHub上の最新データを、自分のPCにダウンロードして同期すること。 |
| **プルリク (Pull Request)** | ブランチで作った変更をメイン軸に合流させる前に、チームへ「確認（レビュー）依頼」を出すこと。 |
| **マージ (Merge)** | プルリクで承認された変更を、メイン軸（`main`）にガチャンと合流させること。 |

---

## 2. 現場配属日に行うこと（最初の一回だけ）

チームに配属され、すでにあるシステムを手元に持ってくる際の手順です。

```bash
# 1. 開発用のフォルダ（ドキュメント等）に移動する
cd 作業したいフォルダのパス

# 2. GitHubからPCへ、リポジトリを丸ごとダウンロードする
git clone [https://github.com/プロジェクトのURL.git](https://github.com/プロジェクトのURL.git)

# 3. ダウンロードしたプロジェクトのフォルダの中に入る
cd リポジトリ名

```

## 3. 毎日の開発サイクル（黄金のフロー）
現場でチケット（タスク）を割り振られ、実際にコードを書いて提出するまでの一連の流れです。この順番を絶対に守ります。
```
# 1. 最新化（※古い状態から作業を始めないため、必ず最初に実行）
git switch main
git pull origin main

# 2. 作業用ブランチ（分身）の作成と移動
git switch -c feature/タスク名

# 3. プログラムを改修する（VSCode等でファイル編集・保存）

# 4. 変更したファイルをセーブ対象としてマークする
git add 変更したファイル名
# （※全ファイルを一気に指定する場合は git add . ）

# 5. セーブする（どんな修正をしたかメッセージを残す）
git commit -m "〇〇の機能を追加"

# 6. 自分の作業ブランチをGitHubへアップロードする
git push -u origin feature/タスク名
```

【この後のブラウザ作業】

- GitHub画面で Compare & pull request を押し、PRを作成してチームにレビュー依頼を出す。

- 承認され、main に Merge されたら、自分のPCのターミナルに戻り 手順1 からやり直す。

## 4.自分ひとりでゼロから環境を作る場合（個人開発用）
今日一番最初にやった「空のフォルダをGit管理して、GitHubに上げる」手順です。
```
# 1. フォルダをGitの管理下（ローカルリポジトリ）にする
git init

# 2. ファイルを作成・保存し、セーブ対象にする
git add .

# 3. セーブする
git commit -m "最初のコミット"

# 4. ブランチ名をmainに変更する
git branch -M main

# 5. GitHub上の空リポジトリと紐づける（URLはGitHub画面からコピペ）
git remote add origin [https://github.com/あなたのユーザー名/リポジトリ名.git](https://github.com/あなたのユーザー名/リポジトリ名.git)

# 6. GitHubにアップロードする
git push -u origin main
```
## 5. 迷子になった時のお助けコマンド
「今自分がどのブランチにいるかわからない」「コミットできているかわからgitない」といった時に、現在の状態を確認する（見るだけ）の安全なコマンドです。
```
# 現在のブランチ名と、未セーブのファイルがあるか確認する（一番よく使います）
git status

# PC内にあるブランチの一覧を見る（*がついているのが現在の場所）
git branch

# 過去のコミット（セーブ）履歴を一覧で見る（終了するには Q キーを押す）
git log
```
